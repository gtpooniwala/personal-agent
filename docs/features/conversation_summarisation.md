# Conversation Summarisation & Condensed Context Window

## Overview
This document describes the conversation summarisation system and the condensed context window logic implemented in the personal agent. The system ensures that long conversations are efficiently summarised, reducing context size while preserving essential information for the agent.

## Motivation
Large language models have a limited context window. As conversations grow, it becomes necessary to condense history without losing important details. Summarisation enables the agent to retain key information and maintain high performance.

## How Summarisation Works
- After each assistant response, the orchestrator triggers an asynchronous background task to summarise the conversation so far.
- The summary is generated by a dedicated summarisation agent/tool.
- The summary is saved as a `SystemMessage` in the conversation history (never as a `ToolMessage`).
- Summaries are not shown to the user but are used internally to manage context.

## Condensed Context Window Logic
When preparing context for the agent:
1. If a summary exists, the agent receives:
   - The latest summary (`SystemMessage`)
   - The 4 most recent non-summary messages before the summary
   - All messages after the summary
2. If no summary exists, the full conversation history is used (up to the context window limit).

This approach ensures the agent has access to recent details and a high-level summary of earlier conversation, optimising both relevance and efficiency.

## Implementation Details
- Summarisation is triggered asynchronously using `asyncio.create_task` in the orchestrator.
- Summaries are generated by the summarisation agent/tool in `backend/orchestrator/tools/summarisation_agent.py`.
- The condensed history logic is implemented in `get_condensed_conversation_history` in `backend/orchestrator/core.py`.
- Summaries are always stored as `SystemMessage` objects to avoid tool_call_id errors.

## Benefits
- Maintains agent performance on long conversations
- Reduces memory and compute requirements
- Ensures recent and summarised context is always available
